# Design Guides

本目錄放置偏設計層、架構層、模型層的 human-facing guides。

## 適合放這裡的文件

- 架構設計方法
- 資料模型與 class 設計準則
- integration / API boundary 設計說明
- 測試設計方法與資料設計參考

## 不適合放這裡的文件

- setup / installation / environment configuration
- troubleshooting / FAQ
- onboarding 或 learning path
- 單一真相規範或 checklist
- AI collaboration / skill / prompt guides

這些內容應分別放到：

- `.dev/guides/implementation-guides/`
- `.dev/guides/learning-guides/`
- `.dev/standards/`
- `.dev/guides/ai-collaboration-guides/`

## 文件索引

- `DATA-CLASS-STANDARDS.md`
  - data class 設計準則與使用界線
- `EZDDD-FRAMEWORK-REFERENCE.md`
  - ezDDD framework 概念對照與參考
- `FRAMEWORK-API-INTEGRATION-GUIDE.md`
  - framework API integration 設計與使用方式
- `PROFILE-BASED-TESTING-GUIDE.md`
  - profile-based testing 設計方式
- `TEST-DATA-PREPARATION-GUIDE.md`
  - 測試資料準備與設計注意事項

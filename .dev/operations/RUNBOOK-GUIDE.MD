# Runbook Guide

本指南定義 `.dev/operations/runbooks/` 下 runbook 文件的最小內容。

## 目的

讓維護者在 incident 或異常行為發生時，快速回答：

- 什麼現象代表需要進入這份 runbook
- 要先檢查哪裡
- 如何區分資料問題、MQ 問題、consumer 問題、contract 問題
- 如何安全恢復與驗證

## 建議目錄

```text
.dev/operations/runbooks/
  README.MD
  <incident-name>.md
```

## 每份 Runbook 最小章節

```markdown
# <Incident Name> Runbook

## Trigger

## Scope

## Symptoms

## Immediate Checks

## Diagnosis Steps

## Recovery Actions

## Verification

## Escalation / Follow-up
```

## Trigger 要回答

- 什麼訊號會觸發這份 runbook
- 什麼情況不應該使用這份 runbook

## Diagnosis Steps 要回答

- 要檢查哪些 logs / queues / topics / database states
- 先後順序是什麼
- 哪些結果代表不同 root cause

## Recovery Actions 要回答

- 哪些動作是安全的
- 哪些動作需要人工批准
- 哪些動作可能造成 replay / duplicate side effects

## Verification 要回答

- 怎麼確認 incident 已解除
- 哪些 event / records / states 應恢復正常
- 是否需要補資料或建立後續 refactor task

# E2E Test Specs

This directory contains test specifications for end-to-end journeys.

Suitable content:

- Complete business journeys beginning at an API or UI entry point
- Verification spanning adapters, application logic, messaging, and persistence
- Scenarios prioritizing user-observable results

Unsuitable content:

- Test intent for a single use-case handler
- Verification of a single aggregate invariant
- Integration verification focused only on a repository or MQ contract

## 概述

本指南定義 .NET 範本與專案程式碼之間的同步規範，確保範本反映最新設計模式與最佳實踐。

## 範本分類架構

```
examples/
├── core/               # 核心模式（穩定）
│   ├── aggregate/
│   ├── value-object/
│   └── repository/
├── patterns/           # 設計模式（定期同步）
│   ├── usecase/
│   ├── projection/
│   ├── mapper/
│   └── controller/
└── advanced/           # 進階模式（頻繁更新）
    ├── contract/
    ├── reactor/
    └── optimization/
```

## 同步原則
- 範本必須符合 DDD/CA/CQRS 與 Event Sourcing 規範
- 只同步已經被證實的最佳實踐
- 避免將實驗性改動寫入 core 範本

## 版本控制機制
- 範本變更需記錄在對應的 ADR/Spec
- 重大變更需更新範本版本與說明

## 同步流程
1. 從 production code 中提取穩定模式
2. 更新範本並標註版本
3. 更新 examples/INDEX 與 guides
4. 通知使用者範本更新內容

## 檢查清單
- [ ] 範本內容與現行規範一致
- [ ] 範本路徑正確且可被索引
- [ ] 範本使用說明已更新

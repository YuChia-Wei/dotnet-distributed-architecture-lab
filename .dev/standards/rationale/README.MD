# Standards Rationale

本目錄存放可攜式的模式選擇理由。

它回答的是：

- 為什麼這套知識庫偏好某種模式
- 為什麼不用其他常見替代方案
- 這些規則背後的 tradeoff 是什麼

## 何時放在這裡

- 規則已經落地成標準文件
- 原本理由散落在 ADR
- 這些理由值得跟著 `.dev/` / `.ai/` 一起移植到其他專案

## 與 ADR 的分工

- `ADR`
  - 記錄「這個 repo 當時做了什麼決策」
- `standards/rationale`
  - 記錄「這類規則通常為什麼成立」

因此當某個 ADR 的主要剩餘價值只剩方法論理由時，應優先把理由抽到這裡，再決定是否退役該 ADR。
